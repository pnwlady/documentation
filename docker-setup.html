Starting a new project

  1. Open a JIRA ticket
  2. Document requirements in ticket or confluence if verbose.
  3. Create branch from JIRA ticket by type (bug, feature, etc).
  4. If multiple people are working on the same branch create sub-branches with consistent naming convention with the goal of merging them into feature branch.
  5. Fetch and checkout into that branch.
  6. Work
  7. Git add, commit. When ready to push, checkout to master and pull any updates, check back out to your branch and rebase.
  8. Submit pull request.
  9. Have team member review request, approve and merge. (Reviewer/approver and merger are not the same person.)
  10. Everyone pull down updated master.
  11. Delete local branch.
