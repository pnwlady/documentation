Git Rebase

  1. After your final commit on your branch, checkout to master and pull down any possible remote changes to your local master branch
  2. git checkout master
  3. git pull
  4. Now go back to your branch to rebase (CONNECT-549 –feature-blah)
  5. git checkout CONNECT-549 –feature-blah
  6. git rebase –i master(i=interactive)
  7. Clean up the commit messages
  8. Title & main commit message
  9. rename commit to JIRA ticket ID
  10. summarize commit in one message
  11. Everyone pull down updated master.
  12. fixup or squash the commits
  13. cw to change word
  14. remove pick and replace with f
  15. hit the escape button
  16. use j to go down to the next commit
  17. go through i-iii again
  18. When done push your changes
  19. git push
  20. If there's an error force push
  21. git push –f
  • Set up vim as editor: git config --global core.editor "vim"
  • Set up autosquash: git config --global rebase.autosquash true
  • p=pick (use commit as is)
  • r=reword (use commit and reword)
  • e=edit (use commit and edit)
  • s=squash (use commit but meld into previous commit)
  • f=fixup (use commit but discard commit message)
  • x=exec (run command with shell)
